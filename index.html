Stage · Janv.–Mars 2026

Interopérabilité des données patrimoniales

Le contexte : Lors de mon stage à la Plateforme Humanités Numériques (PHN) de l'UPJV, j'ai été chargé de valoriser numériquement 800 objets patrimoniaux. L'enjeu : rendre ces données accessibles aux grands agrégateurs du Ministère de la Culture.

Le défi technique : Le système existant reposait sur WordPress et le modèle "Dublin Core" — un format plat où les données (mesures, matériaux, etc.) étaient enfouies dans des chaînes PHP sérialisées. Je devais automatiser la conversion vers le standard "LIDO-MC", un modèle XML hiérarchique extrêmement strict.

Ma solution : J'ai développé un moteur de conversion en PHP (script externalisé pour ne pas altérer le cœur de WordPress). J'ai créé un système de "mapping" sémantique reliant les bases locales à des thésaurus internationaux via Linked Data (Wikidata et Getty AAT). Mon algorithme de désérialisation extrayait les données brutes, vérifiait leur validité, puis la bibliothèque XMLWriter générait les flux XML en continu.

Le résultat : Les fichiers XML générés ont passé avec succès les tests rigoureux du validateur XSD du Ministère de la Culture. J'ai prouvé ma capacité à travailler en autonomie sur des normes complexes, tout en collaborant avec une équipe pluridisciplinaire de chercheurs.

Projet personnel · IA & Confidentialité

Ollama & IA Locale

Dans ce projet personnel, j'ai exploré l'intégration de modèles d'intelligence artificielle en local pour garantir la confidentialité des données — une problématique de plus en plus centrale dans le paysage tech.

  • Technologies : Python, Ollama, JSON
  • Réalisation : Création d'une interface capable de détecter ou simuler des émotions, couplée à un système de mémoire persistante basé sur des fichiers JSON structurés
  • Acquis : Compréhension du fonctionnement des LLMs, manipulation de flux de données asynchrones, architecture d'agents conversationnels
  • Pourquoi ce projet : La dépendance aux APIs cloud soulève des questions éthiques légitimes. Ce projet explore la voie des modèles on-premise accessibles à tous

Application de Covoiturage

Projet majeur réalisé dans le cadre d'une SAÉ (Situation d'Apprentissage et d'Évaluation) en BUT2 — de la conception du modèle de données jusqu'au déploiement de l'interface.

  • Technologies : PHP, SQL (Oracle), HTML/CSS/JS

Projet académique · Algorithmique

Moteur "Jeu de la Vie"

Le célèbre automate cellulaire imaginé par John Horton Conway — recréé, optimisé et poussé dans ses retranchements.

  • Technologies : C++ / Java selon les versions
  • Réalisation : Programmation d'un moteur performant capable de calculer l'évolution de milliers de cellules simultanément sans perte de framerate, avec visualisation en temps réel
  • Acquis : Rigueur algorithmique, gestion fine de la mémoire, optimisation de boucles critiques, trade-offs entre lisibilité et performance
  • Optimisations notables : Implémentation d'un algorithme différentiel ne recalculant que les cellules modifiées à chaque cycle, réduisant drastiquement la charge CPU
