01 / 05

Le Contexte & L'Enjeu

Stage de 8 semaines au sein de la Plateforme Humanités Numériques (PHN) de l'UPJV. La mission consistait à valoriser environ 800 objets patrimoniaux hétérogènes du site Phileas.

L'objectif majeur : sortir les collections de leur "isolement numérique" en les rendant interopérables et conformes aux exigences du Ministère de la Culture pour être moissonnées par les agrégateurs nationaux, comme Ribambelle.

UPJV Patrimoine Numérique Moissonnage de données
Schéma de base simplifié - Phileas WP

02 / 05

Le défi de la base WordPress

Le système historique reposait sur un modèle Dublin Core "plat" de seulement 15 éléments de base. Ce format manquait de granularité (par exemple, impossibilité de différencier une "date de création" d'une "date de découverte").

Du côté de la base de données (MariaDB), les notices étaient stockées en Custom Post Types. Une immense contrainte technique s'est imposée : les métadonnées étaient enfouies dans la table wp_postmeta sous forme de chaînes PHP sérialisées, les rendant inexploitables par des requêtes SQL classiques.

Problématique : Comment assurer l'interopérabilité des collections en automatisant la conversion d'un modèle de données simple et sérialisé vers le standard LIDO-MC hiérarchique complexe ?

Architecture du script PHP externalisé

03 / 05

Ingénierie du Moteur PHP

J'ai développé un script PHP externalisé agissant directement avec le cœur de WordPress pour ne pas perturber l'environnement de production.

J'ai créé des fonctions algorithmiques (comme getSerializedValue) pour désérialiser et extraire proprement les données brutes.

Pour la génération, l'utilisation de SimpleXML aurait saturé la mémoire. J'ai donc implémenté la bibliothèque XMLWriter pour générer l'arborescence XML en flux continu.

Norme W3C - Web sémantique

04 / 05

Web Sémantique & Linked Data

Générer des balises XML ne suffisait pas : il fallait lier les données à des concepts universels. Le standard LIDO-MC est centré sur l'événement (Création, Acquisition...).

J'ai conçu et intégré une table META_CONCEPTS permettant de réaliser un "mapping" sémantique]. L'algorithme attribue désormais automatiquement des URI internationales issues de grands thésaurus (Getty AAT, Wikidata) via la fonction getEventURI.

XMLWriter XSD v1.1 Getty AAT Wikidata URI
Validation du fichier XSD

05 / 05

Validation & Impact

Le principal défi XML était l'ordre strict des balises imposé par le schéma XSD v1.1. L'algorithme vérifie dynamiquement (via hasValue) la présence de données avant d'ouvrir des nœuds parents, évitant ainsi de générer des balises vides qui invalideraient le flux.

Le fichier final a passé avec succès les validateurs XSD officiels du Ministère de la Culture. L'UPJV dispose désormais d'une solution sur-mesure, garantissant une totale souveraineté technique sur ses données.

01 / 05

Répondre à un besoin étudiant

L'application web "StudyGo" a été conçue pour répondre à des problématiques réelles : les difficultés des étudiants pour venir à l'IUT, le coût de l'essence, et le manque de transports en commun le soir.

La solution proposée est un site de covoiturage exclusif aux étudiants et professeurs de l'IUT, dans le but de favoriser la mobilité durable et réduire l'empreinte carbone. L'application met en relation des conducteurs proposant un trajet et des passagers en recherche.

02 / 05

Gestion de Projet & Agile

Réalisé en équipe de 6 étudiants, ce projet a nécessité une planification rigoureuse avec l'utilisation d'outils collaboratifs comme Trello, GitHub et des réunions régulières (Discord/Présentiel).

Nous avons suivi un cycle de développement complet et itératif : Rédaction du Cahier des Charges (CDC) ➔ Maquettage (PC/Mobile) ➔ Développement Front-end ➔ Développement Back-end ➔ Phase de tests et revues de code.

Collaboration : Le découpage des tâches et la validation collective ont permis d'anticiper les risques de retard et d'optimiser les atouts de chacun.

Modélisation relationnelle des données

03 / 05

Modélisation & Base de Données

La robustesse de l'application repose sur une base de données cloud hébergée sur AlwaysData, gérée via MySQL WorkBench.

Avant d'écrire la moindre ligne de code, nous avons élaboré le dictionnaire de données, le Modèle Conceptuel de Données (MCD) et le Modèle Logique de Données (MLD).

Cette étape était cruciale pour gérer les relations complexes entre les utilisateurs, les véhicules, les annonces de trajets, les réservations et le système d'avis et commentaires.

Développement PHP et Sécurisation

04 / 05

Back-End PHP & Sécurité Appliquée

Le traitement des données côté serveur a été réalisé en PHP natif avec un fort accent sur la sécurité des utilisateurs. L'authentification intègre un algorithme de hachage robuste (SHA-256) pour la protection des mots de passe.

La dynamique de l'application s'appuie sur la gestion stricte des Sessions et des méthodes HTTP (GET/POST) finement paramétrées et validées. Pour contrer les failles courantes (CSRF, XSS, injections SQL), des tokens de sécurité ont été implémentés dans les formulaires critiques.

Projet académique · Algorithmique

Moteur "Jeu de la Vie"

Le célèbre automate cellulaire imaginé par John Horton Conway — recréé, optimisé et poussé dans ses retranchements.

  • Technologies : C++ / Java selon les versions
  • Réalisation : Programmation d'un moteur performant capable de calculer l'évolution de milliers de cellules simultanément sans perte de framerate, avec visualisation en temps réel
  • Acquis : Rigueur algorithmique, gestion fine de la mémoire, optimisation de boucles critiques, trade-offs entre lisibilité et performance
  • Optimisations notables : Implémentation d'un algorithme différentiel ne recalculant que les cellules modifiées à chaque cycle, réduisant drastiquement la charge CPU
